3.2 唔唔啊…分支多了历史好混乱

    git rebase指令对个人开发十分友好，与merge的基本使用方式完全相同，但是rebase指令会重写生成线性历史结构，具体流程与基本指令如下：

  • 找到 feature 相对于 main 的新提交。
  • 临时移除这些提交，将 feature 分支“移到” main 的最新提交之后。
  • 重新逐个应用（replay）那些提交。

需要注意⚠️: 对已经推送到远程的公共分支执行rebase会破坏他人历史，切忌在公共场所随地rebase。
